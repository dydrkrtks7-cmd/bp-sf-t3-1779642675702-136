## 소제목
### PyPI RC3 빌드의 취약성

#### 오픈소스 디펜던시 관리와 PostgreSQL 실행계획 회귀

Apache Superset 2.0 릴리스를 진행하면서, PyPI에 RC3 버전의 Superset이 삭제되었고, 이는 주요한 취약점을 가진 것임을 확인했다. 이 버전은 PostgreSQL에서 사용되는 실행계획에 문제가 있었으며, 이를 통해 악성 코드가 SQL 쿼리를 통해 시스템에 접근할 수 있었다.

### RC3 빌드 삭제의 이유

삭제 결정의 주요 원인 중 하나는 성능 문제였다. SRE 엔지니어들은 특정 마이너 버전에서 디스크 I/O가 증가하여 로그 파일 관리에 문제가 발생했었다. 이로 인해 시스템은 불필요한 작업을 수행하고 있어, 전체 성능 저하를 초래했다.

#### 비교: SRE 엔지니어와 마포 셔츠룸 고객

SRE 엔지니어들은 "특정 버전 업그레이드 후 로그 파일 관리 문제"를 보고했으며, 이는 시스템 성능에 큰 영향을 미쳤다. 마포 셔츠룸의 경우, 고객들의 경험과 비슷한 방식으로 이슈가 발생했다면, 이를 해결하기 위해 SRE 엔지니어와 협력해야 할 것이다.

### 실제 사례: PostgreSQL 실행계획 회귀

삭제된 RC3 버전을 사용할 때 발생하는 문제들은 시스템의 성능 저하를 초래했고, 이로 인해 PostgreSQL에서 중요한 실행계획이 변경되었다. 이를 통해 악성 코드가 SQL 쿼리를 통해 시스템에 접근할 수 있었다.

#### 예외 처리: 실제 적용 사례

삭제된 RC3 버전은 실제 시험 진행을 어렵게 만들었다. SRE 엔지니어들은 성능 문제가 발생했을 때, 어떻게 이를 해결해야 하는지를 직접적으로 경험했다. 마포 셔츠룸 고객들도 이와 같은 경험에서 배울 수 있을 것이다.

### 결론: 단기 효과 vs 장기 효과

SRE 엔지니어들이 RC3 버전의 시험 진행을 어렵게 만들었으므로, 이 문제는 단기적으로 시스템 성능 저하를 초래했다. 그러나 장기적인 관점에서 볼 때, 이러한 문제들을 해결하면 전체 시스템의 보안과 운영 효율성이 향상될 것이다.
